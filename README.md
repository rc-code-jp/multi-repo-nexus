# 複数のリポジトリを持つプロジェクトの親となるリポジトリ

すでにマルチリポジトリで運用されているプロジェクトで、AI のルールなどを親ディレクトリで一括管理したい要望に応える。
環境構築や横断したタスクをこなせる中心的な構成を提供する。
これはそのためのボイラーテンプレート。

## 概要

子フォルダに `.git` ディレクトリを置くことで、そのフォルダを独立した子リポジトリとして扱う。
これによりAIが下層のリポジトリを読み取れる状態を維持しつつ、親リポジトリとは別に管理できる。

補足: Gitは配下に `.git` ディレクトリがあるフォルダを別リポジトリとして扱い、その内部ファイルを親リポジトリの通常の追跡対象にしない。

```
Selected configuration:
  - Codex
```

## 設定方法

子リポジトリの初期化は、各子フォルダで `git init` を実行してください。

親リポジトリから除外したい子フォルダには、`.git` ディレクトリが存在する状態にします。

Codex 用のスキルは `.agents/skills/` 配下に配置します。
